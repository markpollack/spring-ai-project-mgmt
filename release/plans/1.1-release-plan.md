## 💻 Implementation Plan

### Phase 1: Version Type Detection

**Core Logic**:
```python
class ReleaseConfig:
    def get_release_type(self) -> str:
        """Determine release type from version string"""
        if '-M' in self.target_version:
            return 'milestone'
        elif '-RC' in self.target_version:
            return 'rc'
        elif self.target_version.count('.') == 2:
            parts = self.target_version.split('.')
            if parts[2] == '0':
                return 'minor'  # X.Y.0 is a minor release
            else:
                return 'patch'  # X.Y.Z where Z > 0
        return 'unknown'
        
    def validate_version(self) -> bool:
        """Enhanced validation for all release types"""
        # Support X.Y.Z, X.Y.Z-MN, X.Y.Z-RCN
        pattern = r'^\d+\.\d+\.\d+(-M\d+|-RC\d+)?$'
        return bool(re.match(pattern, self.target_version))
```

### Phase 2: Branch Selection Strategy

```python
def determine_default_branch(self) -> str:
    """Auto-select branch based on release type"""
    release_type = self.get_release_type()
    
    if release_type in ['milestone', 'rc', 'minor']:
        return 'main'  # Minor releases and prereleases from main
    elif release_type == 'patch':
        # Patch releases from maintenance branch
        parts = self.target_version.split('.')
        return f"{parts[0]}.{parts[1]}.x"  # e.g., 1.0.x
    else:
        raise ValueError(f"Cannot determine branch for {self.target_version}")
```

**Branch Strategy**:
- **main branch**: 1.1.0-M1, 1.1.0-M2, 1.1.0-RC1, 1.1.0 (GA)
- **maintenance branch**: 1.0.1, 1.0.2 (patches)
- **Override available**: `--branch` parameter always works

### Phase 3: Next Development Version Logic

```python
@property
def next_dev_version(self) -> str:
    """Calculate next development version based on release type"""
    release_type = self.get_release_type()
    
    if release_type == 'milestone':
        # 1.1.0-M1 -> 1.1.0-SNAPSHOT (progressing toward final release)
        base = self.target_version.split('-M')[0]
        return f"{base}-SNAPSHOT"
        
    elif release_type == 'rc':
        # 1.1.0-RC1 -> 1.1.0-SNAPSHOT (preparing for GA)
        base = self.target_version.split('-RC')[0]
        return f"{base}-SNAPSHOT"
        
    elif release_type == 'minor':
        # 1.1.0 -> 1.2.0-SNAPSHOT
        parts = self.target_version.split('.')
        next_minor = str(int(parts[1]) + 1)
        return f"{parts[0]}.{next_minor}.0-SNAPSHOT"
        
    else:  # patch
        # 1.0.1 -> 1.0.2-SNAPSHOT (existing logic)
        parts = self.target_version.split('.')
        next_patch = str(int(parts[2]) + 1)
        return f"{parts[0]}.{parts[1]}.{next_patch}-SNAPSHOT"
```

### Phase 4: Workflow Adjustments

**Milestone Release Workflow**:
```python
def should_skip_post_maven_workflow(self) -> bool:
    """Determine if post-Maven Central steps should be skipped"""
    release_type = self.get_release_type()
    
    # Milestones typically don't need development version updates
    # They are standalone releases in the progression toward GA
    return release_type == 'milestone'
```

**Workflow Steps by Release Type**:

| Release Type | Branch | Post-Maven Steps | Development Version |
|--------------|--------|------------------|-------------------|
| Milestone    | main   | ❌ Skip         | None (standalone) |
| RC           | main   | ✅ Full         | X.Y.Z-SNAPSHOT    |
| Minor (GA)   | main   | ✅ Full         | X.Y+1.0-SNAPSHOT  |
| Patch        | X.Y.x  | ✅ Full         | X.Y.Z+1-SNAPSHOT  |
